Frontend Architect Skill

## Metadata
name: frontend-architect
description: Use for frontend architecture, UI/UX design, routing, components, and client-side data integration

## Overview
This Skill provides specialized frontend architecture expertise for the Todo Full-Stack Web Application. When implementing UI components, routing, user experience, or client-side functionality, apply these architectural principles and best practices. Claude should reference these guidelines whenever creating frontend components to ensure proper architecture, user experience, and maintainability.

## Technology Stack

- Next.js 14+ with App Router
- React 18+ with TypeScript
- Tailwind CSS for styling
- Server Components by default, Client Components only when needed
- Client-side state management

## Architecture Principles

- Use Server Components by default, Client Components only when interactivity is needed
- Create reusable, well-structured components
- Separate presentational and container components
- Implement proper state management
- Follow accessibility best practices (WCAG)

## Component Architecture

- Create reusable, well-structured components
- Use proper TypeScript interfaces for props
- Implement proper error handling and loading states
- Follow consistent naming conventions
- Organize components in a logical directory structure

## API Integration

- Use fetch or axios for API calls
- Implement proper error handling and loading states
- Securely handle JWT tokens in requests
- Follow RESTful API conventions
- Implement proper caching strategies

## UI/UX Standards

- Implement responsive design patterns
- Follow accessibility best practices (ARIA labels, semantic HTML)
- Create consistent design patterns
- Implement proper user feedback mechanisms
- Design for different screen sizes and devices

## When to Apply

Apply these guidelines whenever implementing:
- UI component design
- Routing and navigation
- Client-side data integration
- API integration on the frontend
- Responsive layout creation
- Accessibility improvements
- Frontend architectural decisions

## Code Standards

- Use TypeScript for type safety
- Follow Next.js conventions and best practices
- Implement proper error boundaries
- Use client and server components appropriately
- Write comprehensive tests for critical functionality
- Follow accessibility best practices
